Agents, Skills
& Tools

Πώς το Claude Code αναλύει πολύπλοκη εργασία σε εξειδικευμένους agents που ακολουθούν καθορισμένα skills και χρησιμοποιούν συγκεκριμένα tools — μετατρέποντας μια μεμονωμένη συνεδρία LLM σε μια συντονισμένη ομάδα αυτόνομων εργατών.

01 — Τρία Πρωτόγονα

Τα δομικά στοιχεία

Το σύστημα επεκτάσεων του Claude Code βασίζεται σε τρία συνθέσιμα πρωτόγονα. Καθένα λύνει ένα διαφορετικό πρόβλημα: ποιος κάνει τη δουλειά, πώς τη κάνει, και με τι αλληλεπιδρά.

◉

Agents

Το "ποιος"

Αυτόνομες συνεδρίες Claude με καθορισμένη προσωπικότητα, ρόλο και σύνολο δυνατοτήτων. Κάθε agent έχει μια συγκεκριμένη αποστολή — σχεδίαση, κώδικας, αξιολόγηση, συγγραφή κειμένου — και φέρει τις δικές του οδηγίες, πρόσβαση σε εργαλεία και προτίμηση μοντέλου.

Ορισμός .claude/agents/*.md
Κλήση Από τον ενορχηστρωτή μέσω του εργαλείου Task
Εύρος Δικό του παράθυρο περιβάλλοντος, δικά του εργαλεία
⚙

Skills

Το "πώς"

Επαναχρησιμοποιήσιμες οδηγίες ροής εργασίας που ορίζουν μια πολυφασική διαδικασία. Τα Skills είναι μεθοδολογία — περιγράφουν ποια βήματα πρέπει να ακολουθηθούν, ποια αντικείμενα να παραχθούν και ποια επικύρωση να εκτελεστεί. Φορτώνονται ως πλαίσιο σε όποιον agent τα εκτελεί.

Ορισμός .claude/skills/*/SKILL.md
Κλήση Μέσω /skill-name ή από agents
Εύρος Φορτώνεται στο πλαίσιο του εκτελούντος agent
⚒

Tools

Το "τι"

Συγκεκριμένες ενέργειες που μπορεί να εκτελέσει ένας agent — ανάγνωση αρχείων, εγγραφή κώδικα, εκτέλεση εντολών, αναζήτηση κώδικα, αιτήματα ιστού. Τα Tools είναι η διεπαφή μεταξύ της λογικής του Claude και του πραγματικού κόσμου.

Ορισμός Ενσωματωμένα στο Claude Code + MCP servers
Πρόσβαση Λίστα επιτρεπόμενων εργαλείων ανά agent στο frontmatter
Εύρος Μεμονωμένη ενέργεια, επιστρέφει αποτέλεσμα
02 — Agents

Εξειδικευμένες προσωπικότητες

Ένας agent είναι μια συνεδρία Claude με καθορισμένη ταυτότητα. Το YAML frontmatter στο αρχείο .md του δηλώνει το όνομα, την περιγραφή, το μοντέλο, τα διαθέσιμα εργαλεία και τα φορτωμένα skills. Το σώμα markdown παρέχει τις οδηγίες του — ρόλο, συμπεριφορά ομάδας, φόρτωση πλαισίου, κριτήρια επικύρωσης και κατευθυντήριες αρχές.

Δομή Ορισμού Agent
--- YAML frontmatter --- name: brand-designer description: "Μεταφράζει την ταυτότητα brand σε συστήματα σχεδίασης..." tools: Read, Write, Grep, Glob, WebSearch model: opus memory: project skills: - design-system - frontend-design - theme-factory --- # Σώμα Markdown = οδηγίες agent ## Role ← Τι κάνει αυτός ο agent ## Team Role ← Πώς αλληλεπιδρά με άλλους agents ## Context ← Ποια έγγραφα να διαβάσει πριν ξεκινήσει ## Validation ← Πύλες ποιότητας που πρέπει να περάσει ο agent ## Principles ← Κατευθυντήριες γραμμές λήψης αποφάσεων

Agents σε αυτό το έργο

brand-designer
Μεταφράζει το όραμα του brand σε υλοποιήσιμο σύστημα σχεδίασης. Δημιουργεί παλέτες χρωμάτων, τυπογραφία, κλίμακες αποστάσεων και μοτίβα στοιχείων.
opus Read Write WebSearch design-system frontend-design
▶ λεπτομέρειες
Παράγει: DESIGN_SYSTEM.md, showcase.html, SCREENS.md
Επικυρώνει: Αντίθεση WCAG, swap test, έλεγχος αντι-μοτίβων, μοτίβα πλατφόρμας
ΔΕΝ κάνει: Υλοποίηση στοιχείων UI — τροφοδοτεί προδιαγραφές στον expert-developer
expert-developer
Γράφει κώδικα παραγωγής. Ορίζει συμβόλαια API, μοντέλα δεδομένων και μοτίβα υλοποίησης. Συνεργάζεται με QA για TDD.
opus Read Write Edit Bash code-style live-verification
▶ λεπτομέρειες
Παράγει: Κώδικα παραγωγής, ολοκληρώσεις API, υλοποιήσεις στοιχείων
Διαδικασία: Ανάγνωση απαιτήσεων → Επισκόπηση κώδικα → Ενημέρωση προδιαγραφών → Αναμονή αποτυχημένων δοκιμών → Υλοποίηση → Ζωντανή επαλήθευση
Hooks: Αυτόματη μορφοποίηση στο Edit/Write (prettier, ruff, gofmt)
copywriter
Συγγραφέας κειμένων με εστίαση στη μετατροπή. Γράφει τίτλους, κυρίως κείμενο, CTAs και μικροκείμενα για παραδοτέα που απευθύνονται στον πελάτη.
sonnet μέλος creative-studio
▶ λεπτομέρειες
Παράγει: copy.md — όλο το κείμενο για κάθε ενότητα
Φιλοσοφία: Άμεση απάντηση. Ξεκίνα με το πρόβλημα. Πρόταση αξίας μέχρι τη λέξη 50. Χωρίς γέμισμα.
Αποτέλεσμα: Δεδομένα περιεχομένου JSON που αντιστοιχούν σε σχήματα στοιχείων
customer
Αξιολογεί τα αποτελέσματα ως ο πελάτης-στόχος. Βαθμολογεί σε 5 διαστάσεις. Πάντα βρίσκει λόγους που ένας υποψήφιος θα αποχωρούσε. Δεν εκδίδει ποτέ PASS.
opus Read Grep αξιολογητής
▶ λεπτομέρειες
Διαστάσεις: Οπτική, Κείμενο, Ροή, Πεποίθηση, Πειθώ
Σοβαρότητα: P1 (αιτία αποχώρησης), P2 (δισταγμός), P3 (διάβρωση εμπιστοσύνης)
Ετυμηγορία: Πάντα REVIEW. Δεν υπάρχει PASS. Κάθε αποτέλεσμα έχει τριβή.
03 — Skills

Επαναχρησιμοποιήσιμες ροές εργασίας

Ένα skill είναι ένα αρχείο SKILL.md που ορίζει μια πολυβηματική διαδικασία. Τα Skills δεν εκτελούνται μόνα τους — φορτώνονται ως οδηγίες στον agent που τα εκτελεί. Ένα skill μπορεί να ενορχηστρώσει πολλαπλούς agents, να ορίσει φάσεις, να καθορίσει αντικείμενα και να περιλαμβάνει ελέγχους επικύρωσης. Οι χρήστες καλούν skills με /skill-name.

Δομή Ορισμού Skill
--- YAML frontmatter --- name: creative-studio description: "Ενορχηστρώνει 3 παράλληλες δημιουργικές ομάδες..." --- # Σώμα Markdown = ορισμός ροής εργασίας ## Input ← Ποια ορίσματα δέχεται το skill ## Workflow ← Βήματα εκτέλεσης ανά φάση Phase 1: Σχεδίαση (δημιουργία 3 agents brand-designer) Phase 2: Κείμενο (δημιουργία 3 agents copywriter) Phase 3: Κατασκευή (δημιουργία 3 agents expert-developer) Phase 4: Αξιολόγηση (δημιουργία 1 agent customer) Phase 5: Επανάληψη ή Αποστολή ## Rules ← Αυστηροί περιορισμοί κατά την εκτέλεση ## References ← Υποστηρικτικά αρχεία (πρότυπα, prompts)

Δύο τύποι skills

Skills Μεθοδολογίας

Φορτώνονται ως πλαίσιο

Παρέχουν οδηγίες και κατευθυντήριες γραμμές που διαμορφώνουν τον τρόπο λειτουργίας ενός agent. Δεν δημιουργούν υπο-agents — αλλάζουν τη συμπεριφορά του εκτελούντος agent προσθέτοντας περιορισμούς, λίστες ελέγχου και πρότυπα ποιότητας.

Παραδ. frontend-design, brainstorming, code-style
Agents 0 δημιουργούνται — φορτώνονται στον εκτελεστή

Skills Ενορχήστρωσης

Συντονίζουν agents

Ορίζουν ροές εργασίας πολλαπλών agents που δημιουργούν εξειδικευμένους υπο-agents παράλληλα ή διαδοχικά. Ελέγχουν τη ροή — ποιοι agents θα δημιουργηθούν, πότε, με ποιες οδηγίες και πώς θα συνθέσουν τα αποτελέσματα.

Παραδ. creative-studio, agent-team, feature-frontend
Agents Πολλαπλοί δημιουργούνται ανά φάση

Skills σε αυτό το έργο

brainstorming
Εξερεύνηση πρόθεσης και σχεδίασης πριν την υλοποίηση. Μία ερώτηση τη φορά.
μεθοδολογία
design-system
Δημιουργία πλήρων design tokens από ταυτότητα brand. 7 φάσεις, 7 επικυρώσεις.
ενορχήστρωση
frontend-design
Ξεχωριστό UI επιπέδου παραγωγής. 5 φάσεις, 6 αυτο-έλεγχοι.
μεθοδολογία
creative-studio
3 ανταγωνιστικές ομάδες → αξιολόγηση πελάτη → ο νικητής επαναλαμβάνει → αποστολή.
ενορχήστρωση
agent-team
Συντονισμός παράλληλων συνεδριών Claude. Κατασκευή χαρακτηριστικών, αξιολόγηση πολλαπλών οπτικών, ανταγωνιστικές υποθέσεις.
ενορχήστρωση
feature-frontend
Pipeline frontend με προτεραιότητα σχεδίασης: brainstorm → σχεδίαση → TDD → επαλήθευση → αξιολόγηση.
ενορχήστρωση
agent-browser
Αυτοματοποίηση αλληλεπιδράσεων browser για δοκιμές ιστού, στιγμιότυπα οθόνης και επαλήθευση.
μεθοδολογία
skill-creator
Μετα-skill: οδηγός για τη δημιουργία νέων skills που επεκτείνουν τις δυνατότητες του Claude.
μεθοδολογία
agent-config-generator
Δημιουργία CLAUDE.md και AGENTS.md ακολουθώντας βέλτιστες πρακτικές.
μεθοδολογία
04 — Tools

Συγκεκριμένες ενέργειες

Τα Tools είναι τα χέρια και τα μάτια ενός agent. Κάθε εργαλείο εκτελεί μία συγκεκριμένη ενέργεια — ανάγνωση αρχείου, εκτέλεση εντολής, αναζήτηση κώδικα, επεξεργασία κειμένου. Οι agents δηλώνουν σε ποια εργαλεία έχουν πρόσβαση στο frontmatter τους. Ο ενορχηστρωτής αποφασίζει ποια εργαλεία θα παραχωρήσει σε κάθε υπο-agent βάσει του ρόλου του.

Λειτουργίες Αρχείων
Read — Προβολή περιεχομένων αρχείων, εικόνων, PDF
Write — Δημιουργία ή αντικατάσταση αρχείων
Edit — Ακριβείς αντικαταστάσεις συμβολοσειρών
Glob — Εύρεση αρχείων βάσει μοτίβου
Grep — Αναζήτηση περιεχομένων αρχείων (regex)
Εκτέλεση
Bash — Εκτέλεση εντολών κελύφους (git, npm)
Task — Δημιουργία υπο-agent για σύνθετη εργασία
Skill — Κλήση ενός skill με όνομα
NotebookEdit — Επεξεργασία Jupyter notebooks
Ιστός & Αλληλεπίδραση
WebSearch — Αναζήτηση στο διαδίκτυο
WebFetch — Λήψη και επεξεργασία URL
AskUserQuestion — Διαδραστικά ερωτήματα
MCP Tools — Ολοκληρώσεις εξωτερικών servers
Έλεγχος Πρόσβασης Εργαλείων
// Το frontmatter του Agent περιορίζει ποια εργαλεία μπορεί να χρησιμοποιήσει κάθε agent brand-designer: Read, Write, Grep, Glob, WebSearch // Χωρίς Bash, χωρίς Edit expert-developer: Read, Write, Edit, Grep, Glob, Bash // Πλήρης πρόσβαση κώδικα customer: Read, Grep, Glob // Αξιολογητής μόνο ανάγνωσης copywriter: (όλα τα εργαλεία μέσω ενορχηστρωτή) // Πλήρης πρόσβαση για περιεχόμενο // Ο ενορχηστρωτής (Claude) έχει πρόσβαση σε ΟΛΑ τα εργαλεία // Οι υπο-agents λαμβάνουν περιορισμένη πρόσβαση βάσει ρόλου

Αρχή Ελάχιστων Δικαιωμάτων

Οι agents λαμβάνουν μόνο τα εργαλεία που χρειάζονται. Ένας αξιολογητής customer δεν έχει Write ή Bash — μπορεί μόνο να διαβάζει και να αναλύει, αποτρέποντας τυχαίες τροποποιήσεις.

Τα Hooks Επεκτείνουν τα Tools

Οι agents μπορούν να ορίσουν hooks που ενεργοποιούνται μετά τη χρήση εργαλείου. Ο expert-developer μορφοποιεί αυτόματα τον κώδικα μετά από κάθε κλήση Edit ή Write μέσω ενός PostToolUse hook.

05 — Αρχιτεκτονική

Πώς συντίθενται

Τα τρία πρωτόγονα σχηματίζουν μια στρωματοποιημένη αρχιτεκτονική. Ο χρήστης επικοινωνεί με τον ενορχηστρωτή (κύρια συνεδρία Claude). Ο ενορχηστρωτής φορτώνει skills για οδηγίες ροής εργασίας, δημιουργεί agents για εξειδικευμένη εργασία, και κάθε agent χρησιμοποιεί tools για να αλληλεπιδράσει με τον κόσμο.

06 — Ενορχήστρωση

Μοτίβα εκτέλεσης

Ο τρόπος που ο ενορχηστρωτής συντονίζει τους agents εξαρτάται από το skill που εκτελείται. Τρία μοτίβα αναδύονται: μονός agent (ένας ειδικός κάνει τα πάντα), pipeline (οι agents εκτελούν διαδοχικά), και παράλληλες ομάδες (πολλαπλοί agents ανταγωνίζονται ή συνεργάζονται ταυτόχρονα).

Μονός Agent

Γραμμική εκτέλεση

Ο ενορχηστρωτής φορτώνει ένα skill μεθοδολογίας και εκτελεί όλες τις φάσεις μόνος του. Δεν δημιουργούνται υπο-agents.

Χρήστης → Ενορχηστρωτής + skill → Αποτέλεσμα
Χρησιμοποιείται από: frontend-design, brainstorming

Pipeline

Διαδοχικοί agents

Οι agents εκτελούν διαδοχικά, κάθε ένας καταναλώνοντας το αποτέλεσμα του προηγούμενου agent. Ο ενορχηστρωτής διαχειρίζεται τις μεταβιβάσεις.

Α → αντικείμενο → Β → αντικείμενο → Γ
Χρησιμοποιείται από: design-system, feature-frontend

Παράλληλες Ομάδες

Ανταγωνιστικοί agents

Πολλαπλοί agents δουλεύουν το ίδιο brief ταυτόχρονα από διαφορετικές οπτικές γωνίες. Ένας αξιολογητής επιλέγει τον νικητή.

Α \ Β —→ αξιολογητής → ο νικητής επαναλαμβάνει Γ /
Χρησιμοποιείται από: creative-studio, agent-team
07 — Ζωντανό Παράδειγμα

Το Creative Studio σε δράση

Το skill creative-studio επιδεικνύει και τα τρία πρωτόγονα να συνεργάζονται. Ένας χρήστης πληκτρολογεί /creative-studio "landing page" και το σύστημα ενορχηστρώνει 10+ δημιουργίες agents σε 5 φάσεις.

0

Προετοιμασία Brief

Ο ενορχηστρωτής διαβάζει τα έγγραφα του έργου, επιλέγει 3 δημιουργικές οπτικές γωνίες, γράφει δομημένο brief στον δίσκο. Δεν δημιουργούνται agents — ο ενορχηστρωτής κάνει αυτό χρησιμοποιώντας τα δικά του εργαλεία.

skill: creative-studio Read Write
1

Σχεδίαση (3 ομάδες παράλληλα)

Δημιουργεί 3 agents brand-designer ταυτόχρονα μέσω 3 κλήσεων Task σε ένα μήνυμα. Κάθε ένας διαβάζει το brief με διαφορετική δημιουργική οπτική γωνία και παράγει μια προδιαγραφή σχεδίασης. Κάθε ένας φέρει το skill μεθοδολογίας frontend-design.

agent: brand-designer ×3 skill: frontend-design Task Read Write
2

Κείμενο (3 ομάδες παράλληλα)

Δημιουργεί 3 agents copywriter. Κάθε ένας διαβάζει τις προδιαγραφές σχεδίασης της ομάδας του και γράφει όλο το κείμενο — τίτλους, κυρίως κείμενο, CTAs, μικροκείμενα. Το κείμενο ταιριάζει με τη δημιουργική οπτική γωνία.

agent: copywriter ×3 Task Read Write
3

Κατασκευή (3 ομάδες παράλληλα)

Δημιουργεί 3 agents expert-developer. Κάθε ένας διαβάζει τις προδιαγραφές σχεδίασης + κείμενο και υλοποιεί κώδικα έτοιμο για παραγωγή. Το PostToolUse hook μορφοποιεί αυτόματα σε κάθε εγγραφή.

agent: expert-developer ×3 Task Write Edit Bash
4

Αξιολόγηση Πελάτη

Δημιουργεί 1 agent customer. Διαβάζει τα αποτελέσματα και των 3 ομάδων, βαθμολογεί κάθε ένα σε 5 διαστάσεις, επιλέγει τον νικητή, παρέχει εφαρμόσιμη ανατροφοδότηση. Αν ο μέσος όρος βαθμολογίας < 8, η νικήτρια ομάδα επαναλαμβάνει (επιστροφή στη Φάση 1 με 1 ομάδα).

agent: customer ×1 Read Grep Glob

Τι μόλις συνέβη

10+
agents δημιουργήθηκαν σε 5 φάσεις
4
skills φορτώθηκαν (creative-studio, frontend-design, code-style, live-verification)
7
διαφορετικά tools χρησιμοποιήθηκαν (Read, Write, Edit, Bash, Grep, Glob, Task)
08 — Δομή Αρχείων

Πού βρίσκονται όλα

Όλη η διαμόρφωση βρίσκεται κάτω από τον κατάλογο .claude/ στη ρίζα του έργου. Αυτός ο κατάλογος ελέγχεται με σύστημα εκδόσεων και μοιράζεται σε όλη την ομάδα. Το CLAUDE.md στη ρίζα παρέχει οδηγίες επιπέδου έργου που κληρονομούν όλοι οι agents.

project/ CLAUDE.md # Οδηγίες επιπέδου έργου (φορτώνονται πάντα) .claude/ agents/ # Ορισμοί agents brand-designer.md # YAML frontmatter + οδηγίες expert-developer.md customer.md copywriter.md skills/ # Ορισμοί skills brainstorming/ SKILL.md # Ορισμός ροής εργασίας creative-studio/ SKILL.md references/ # Υποστηρικτικά αρχεία creative-angles.md brief-template.md team-handoff-prompts.md design-system/ SKILL.md references/ token-generation-guide.md design-imperatives.md desktop-patterns.md mobile-patterns.md frontend-design/ SKILL.md agent-team/ SKILL.md references/ team-patterns.md conflict-avoidance.md rules/ # Πρόσθετοι κανόνες (φορτώνονται πάντα) coding-rules.md docs/ DESIGN/ DESIGN_SYSTEM.md # Δημιουργήθηκε από το skill design-system showcase.html # Αντικείμενο οπτικής επικύρωσης SCREENS.md # Προδιαγραφές ανά οθόνη

Βασικές γνώσεις

Το νοητικό μοντέλο για τη δημιουργία αποτελεσματικών ροών εργασίας Claude Code.

Οι Agents είναι ειδικοί, όχι γενικοί

Κάθε agent έχει στενό ρόλο με περιορισμένα εργαλεία. Ένας brand-designer δεν μπορεί να εκτελέσει εντολές κελύφους. Ένας customer δεν μπορεί να τροποποιήσει αρχεία. Οι περιορισμοί παράγουν καλύτερα αποτελέσματα από την ελευθερία.

Τα Skills είναι το πραγματικό επίπεδο ενορχήστρωσης

Τα Skills ορίζουν τη ροή εργασίας — ποιους agents να δημιουργήσουν, με ποια σειρά, με ποιες εισόδους. Ο ενορχηστρωτής ακολουθεί τις οδηγίες του skill. Σύνθετη συμπεριφορά αναδύεται από απλή σύνθεση skills.

Τα Tools είναι το σύνορο με την πραγματικότητα

Ό,τι "κάνει" ένας agent συμβαίνει μέσω κλήσης εργαλείου. Ανάγνωση αρχείου, εγγραφή κώδικα, εκτέλεση δοκιμών, δημιουργία άλλου agent — όλα κλήσεις εργαλείων. Το επίπεδο tools είναι εκεί όπου ο συλλογισμός συναντά τον πραγματικό κόσμο.

Τα αντικείμενα είναι ο μηχανισμός μεταβίβασης

Οι agents επικοινωνούν γράφοντας αρχεία. Ο brand-designer γράφει design-spec.md. Ο copywriter το διαβάζει και γράφει copy.md. Ο expert-developer διαβάζει και τα δύο και γράφει κώδικα. Τα αρχεία είναι η κοινή μνήμη.

Οι παράλληλες ομάδες νικούν τη διαδοχική βελτίωση

Τρεις ομάδες που ανταγωνίζονται στο ίδιο brief με διαφορετικές οπτικές γωνίες παράγουν καλύτερα αποτελέσματα από μία ομάδα που επαναλαμβάνει. Ο ανταγωνισμός αναδεικνύει ιδέες που η συναίνεση χάνει.

Το CLAUDE.md είναι το σύνταγμα

Κάθε agent κληρονομεί τις οδηγίες του CLAUDE.md. Πρότυπα επιπέδου έργου, πύλες ποιότητας και φιλοσοφία μεταφέρονται σε όλους τους υπο-agents αυτόματα. Μία πηγή αλήθειας για όλο το σύστημα.

Πρώτα σχεδιασμός, μετά ομάδα

Λανθασμένη ανάλυση εργασιών κοστίζει 500k+ tokens. Πάντα κάνε brainstorm και σχεδίασε πριν δημιουργήσεις μια ομάδα. Όρισε κοινά συμβόλαια πριν κανένας agent αρχίσει να γράφει κώδικα.

Τα πάντα είναι markdown

Agents, skills, κανόνες, συστήματα σχεδίασης, προδιαγραφές — όλα ορίζονται σε αρχεία markdown. Χωρίς ειδικό runtime, χωρίς μεταγλώττιση, χωρίς ανάπτυξη. Ελεγχόμενα με εκδόσεις, αναγνώσιμα από ανθρώπους, συνθέσιμα.
