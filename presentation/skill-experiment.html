Why this matters

The Experiment

Skills are the building blocks of agent capability. But not all skills are created equal. Some load methodology. Some orchestrate pipelines. Some spawn competing agents and iterate until quality gates pass.

This experiment isolates the variable: skill architecture. Everything else stays constant — same model, same prompt, same design system, same brand guidelines.

Control Variables

  • Identical prompt for all 3 runs
  • Same design system tokens
  • Same brand guidelines
  • Same business docs (ICP, Product)
  • Same model (Claude)

Independent Variable

The skill type invoked to execute the prompt. Each adds layers of process, review, and iteration around the same core task.

Dependent Variables

  • Visual design quality
  • Copy quality and brand alignment
  • Design system compliance
  • Mobile responsiveness
  • Overall persuasiveness
The three levels

Three Approaches

From a single methodology injection to a full multi-agent creative studio with customer review loops.

01
Skill

frontend-design

A methodology skill. Loads design thinking principles, anti-patterns, and validation tests into the agent's context. The agent builds the page in a single pass.

  • 1 agent, 1 perspective
  • Methodology as guardrails
  • No separate brainstorming phase
  • No automated review or iteration
02
Workflow Skill

feature-frontend

A sequential pipeline. Checks the design system, brainstorms the design intent, then implements and ships — a 4-step workflow.

  • 1 orchestrator, skill invocations at each step
  • Design system enforced as law
  • Brainstorm mandatory before code
  • Design doc captures decisions
03
Workflow + Feedback Loops

creative-studio

A multi-agent creative pipeline. Designer and copywriter brainstorm together, a developer builds, a customer persona reviews, and the whole cycle iterates until quality gates pass.

  • 5+ specialized agents
  • Collaborative brainstorming (max 4 rounds)
  • Customer review with 5-dimension scoring
  • Feedback loop: iterate up to 3 times
Identical input

The Prompt

This exact prompt is given to all three approaches. The only difference is which skill processes it.

prompt.mdBuild a 3-section homepage for our orthopaedic surgery practice. Single self-contained index.html file. No stock photo URLs. Read the project docs for context: • docs/DESIGN/BRAND.md • docs/DESIGN/DESIGN_SYSTEM.md • docs/BUSINESS/PRODUCT.md • docs/BUSINESS/ICP.md Save to: presentation/experiment/skill/index.html presentation/experiment/workflow/index.html presentation/experiment/creative-studio/index.html

That's it. No section specs, no CTA details, no card counts, no layout instructions. Each skill decides what "3-section homepage" means — which sections, what copy, how to structure it. The prompt tests the skill's ability to make good decisions autonomously.

Phase Breakdown

  1. Design System CheckRead docs/DESIGN/DESIGN_SYSTEM.md. If populated, use its tokens as hard constraints. If missing, flag to user and optionally invoke the design-system skill first.
  2. BrainstormInvoke the brainstorming skill. Explore user goals, visual hierarchy, interaction states, breakpoints, and how the feature uses design tokens. Write design doc to docs/plans/.
  3. ImplementBuild bottom-up following the brainstorm output: atoms (buttons, inputs) → molecules (cards, groups) → organisms (sections, nav) → pages. All visual choices conform to the design system.
  4. Commit & PRCreate feature branch, commit, push, and open a pull request.

Phase Breakdown

  1. Prepare BriefOrchestrator reads all docs. Writes structured brief to docs/studio/. Captures audience, goals, constraints.
  2. BrainstormBrand designer and copywriter agents in back-and-forth dialogue (max 4 rounds). Converge on a creative direction document — the immutable contract for all subsequent phases.
  3. DesignBrand designer agent creates visual direction: layout, hierarchy, component choices, spacing, color application. References frontend-design methodology.
  4. CopyCopywriter agent writes all content: headlines, body, CTAs, micro-copy. Direct-response focused. Conversion-optimized.
  5. BuildExpert developer agent implements the design + copy into code. Orchestrator verifies in browser via agent-browser.
  6. Customer Review & IterateCustomer persona agent scores across 5 dimensions. If average < 8 and round < 3: loop back to Design with targeted feedback. Otherwise: ship.

Conclusions

Key Takeaways

Skills are guardrails, not guarantees

A methodology skill improves the floor — it prevents common mistakes. But it can't add perspectives the agent doesn't have. The output is only as good as a single agent's interpretation.

Thinking before building changes everything

The workflow skill's power is the mandatory brainstorm. Forcing the agent to articulate intent, hierarchy, and states before writing code means better decisions — even though it's the same agent doing both the thinking and the building.

Feedback loops are the multiplier

The creative studio's power isn't the number of agents — it's the iteration cycle. A customer persona finding weak copy in round 1 means the copywriter rewrites it in round 2. Quality improves non-linearly.

Match complexity to stakes

Not every page needs a creative studio. Internal tools benefit from workflow skills. High-stakes landing pages justify the 10x cost of multi-agent iteration. Choose the architecture that matches the deliverable's impact.
